1
Export CHOP standards as a Figma variable collection
Figma Variables ▼

In Figma, open your design file and go to Edit → Variables (or press ⌥V). Create a new collection called CHOP 2026. Add one Color variable per token from the table above — e.g. color/primary/navy = #005587, color/data/pink = #E1176B. Add Number variables for spacing: spacing/visual-padding = 16, layout/sidebar-width = 58.

// Figma variable naming convention color/brand/navy → #005587 color/brand/cyan → #41B6E6 color/data/1-navy → #005587 color/data/2-cyan → #41B6E6 color/data/3-pink → #E1176B color/data/4-green → #6F972B color/text/primary → #1c2024 color/text/secondary → #60646c color/border/default → #cdced6 color/bg/page → #fcfcfd number/sidebar/width → 58 number/padding/visual → 16 number/canvas/width → 1260 number/canvas/height → 720
2
Set up the CHOP frame as a Figma template
Frame setup ▼

Create a new Frame at exactly 1260 × 720 px (matching the .pbix canvas). Name it CHOP / Sample Page Layout. Inside it, place a Rectangle at x=0, y=0, w=58, h=720 filled with color/brand/navy — this is the sidebar. Add a Text frame at x=75, y=16, w=960, h=33 with font Segoe UI Bold 14px, color color/text/primary for the page title.

3
Create reusable CHOP components
Components ▼

Build a Figma component library matching each .pbix visual type. For the KPI Card: a 260×114 frame, fill color/bg/page, border color/border/default 0.5px, containing a label text (Arial 10.5px, color/text/secondary) and value text (Arial Bold 36px, color/data/3-pink). For Slicers: use Calibri 10px, border color/brand/cyan. Create variants for each chart type.

// KPI Card component structure Component "CHOP/KPI Card" [260×114] fill: color/bg/page (#fcfcfd) border: 0.5px color/border/default (#cdced6) padding: 16px ├── Text "label" Arial 10.5px color/text/secondary position: above └── Text "value" Arial Bold 36px color/data/3-pink text-align: center // Slicer component Component "CHOP/Slicer/Dropdown" [auto × 54] fill: color/bg/page border: 0.5px color/brand/cyan padding: 0 ├── Header Calibri Bold 10px color/brand/navy └── Item Calibri Regular 10px color/text/primary bg: #FFFFFF // Chart placeholder Component "CHOP/Chart/Placeholder" [auto × auto] fill: color/bg/page border: 0.5px color/border/default padding: 16px └── Label Segoe UI 9px color/text/secondary
4
Import your Qlik screenshot into the CHOP frame
Screenshot overlay ▼

Paste your Qlik screenshot into Figma (⌘V). Place it on a separate layer called ref/qlik-screenshot. Set its opacity to 40% so it becomes a reference underlay. Resize it to 1260×720 to align with your CHOP frame. Now trace over it: place CHOP components (KPI cards, slicers, chart placeholders) at the positions that match the Qlik visual positions, snapping to the CHOP pixel grid.

Layers panel structure: ▸ Frame "CHOP/Sample Page Layout" ← your working frame ├── [all CHOP components above] └── Image "ref/qlik-screenshot" opacity: 40% blend mode: Normal x=58 y=0 w=1202 h=720 ← inside sidebar // Tip: use Figma's "Compare" view (⌥ drag) to // toggle between Qlik original and CHOP redesign
5
Apply CHOP theme via Figma API (automated)
Figma API ▼

Use the Figma REST API to programmatically apply CHOP colors to every node in your file. First fetch the file's node tree, then iterate over nodes matching visual types (fill rectangles, text layers) and PATCH their fill colors to the CHOP palette. This is the fastest way to recolor an entire screenshot tracing in one pass. See the "Figma API code" tab for the full script.

// High-level API flow 1. GET /files/{fileKey} → full node tree 2. Filter nodes by type (RECTANGLE, TEXT) 3. Map each node's current color → nearest CHOP color 4. PATCH /files/{fileKey}/nodes → apply new fills 5. GET /files/{fileKey}/images → export result
6
Export back to Power BI via Figma → .pbit
Round-trip ▼

Once your Figma design uses CHOP standards throughout, use the Figma API to read the final frame layout (x, y, w, h for each visual placeholder) and feed those pixel coordinates directly into your Power BI template JSON spec. This ensures the Power BI report canvas matches the Figma design pixel-for-pixel, completing the round-trip from Qlik screenshot → CHOP Figma design → Power BI report.

// Round-trip: Figma frame positions → Power BI spec const figmaFrame = await fetch('/files/KEY/nodes?ids=FRAME_ID'); const nodes = figmaFrame.document.children; const pbiVisuals = nodes.map(node => ({ type: mapFigmaTypeToPBI(node.name), // KPI Card → cardVisual x: Math.round(node.absoluteBoundingBox.x - 58), // offset sidebar y: Math.round(node.absoluteBoundingBox.y), width: Math.round(node.absoluteBoundingBox.width), height: Math.round(node.absoluteBoundingBox.height), theme: 'CHOP_20264117488249465352.json' }));
